# Jrss — локальный анти-чит инструментарий для screenshare-проверок

**Автор: Lain** · Discord: rewakura

Jrss — это набор сканеров для выявлениячитов в Minecraft Java Edition. Всё работает локально на машине оператора, без отправки данных на сервер. Ни один нейросетевой анти-чит не обнаружит запуск Jrss, потому что его нет в процессах проверяемого.

## Возможности

### 1. YARA File / Disk Scan
Сканирование файлов и дисков по YARA-правилам. Обходит все локальные диски или указанный путь. Пропускает Authenticode-подписанные файлы для ускорения. Находит известные чит-клиенты по сигнатурам в файловой системе.

### 2. Java Memory Signature Scan
Сканирование памяти javaw.exe/javaw.exe по базе сигнатур через Aho-Corasick-автомат. Ищет строковые индикаторы в committed-памяти процесса. Работает на уровне ReadProcessMemory, не оставляет следов в целевой JVM.

### 3. JVMTI / Loaded DLL Inspector
Проверка загруженных DLL через CreateToolhelp32Snapshot. Выявляет подозрительные библиотеки, включая JVMTI-агенты, инжекторы и обфусцированные Native-загрузчики.

### 4. Launch Script Finder
Поиск вредоносных и байпасс скриптов связанных с Minecraft. Находит аномалии в JVM-аргументах, пути к чит-клиентам, инжекторы через Java-агенты и другие техники запуска.

### 5. Bypass Scanner
Проверка системных обходов, используемых читами:
- DNS-подмены
- Блокировка серверов через firewall
- Подозрительные сервисы Windows
- Системные утилиты, используемые для сокрытия процессов

### 6. External Java Class Dumper
Дамп загруженных классов через штатный Java Attach API (как VisualVM/jcmd). Без инжекта — официальный механизм JVM. Извлекает байткод всех классов работающего Java-процесса для ручного анализа. (чтобы типа с платным читом забанили нах) - (проверенно и работает хаххаа)

### 7. RWX Memory Scanner
Поиск committed RWX-регионов в памяти Java-процесса. RWX-память — классический индикатор code injection. Детектирует даже те читы, которые не оставляют следов на диске.

## Детектируемые клиенты и софт

| Категория | Обнаруживаемое |
|-----------|---------------|
| **Чит-клиенты** | Vape, Prestige, Novoware, Krypton, Thunderhack, Noname, Grim Client, Slinky |
| **Лоадеры** | Cortex, MHub, SystemDLC, Doomsday, Prestige, Liminar, e.t.c |
| **Индикаторы** | KillAura, Scaffold, Speed, Fly, ESP, Reach, AimAssist, AutoTotem, Inventory Hacks, NoSlowdown |

## Что отличает Jrss

- **Не оставляет следов** — работает на машине оператора, не встраивается в процесс проверяемого
- **Не детектируется** — нет сетевой активности, не создаёт окон, не висит в процессах целевой системы
- **Локальность** — все данные обрабатываются на месте, приватность полная
- **Несколько слоёв** — файловая система, память, DLL, скрипты запуска, системные обходы
- **Лёгкий вес** — single-file бинарник, не требует установки

## Сборка

```powershell
dotnet publish -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true
```

Результат: `bin\Release\net8.0\win-x64\publish\Jrss.exe`

## Использование

1. Запусти `Jrss.exe` от имени администратора (требуется для сканирования памяти и процессов)
2. Выбери модуль из главного меню
3. Если нужно — выбери Java-процесс из списка
4. Анализируй результаты

## Требования

- Windows x64
- .NET 8 runtime (для self-contained билда не требуется)
