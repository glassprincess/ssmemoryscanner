# SSMemScan

Локальный сканер памяти процесса `javaw.exe`/`java.exe` на точные сигнатуры известных
чит-клиентов Minecraft. Ничего никуда не отправляет — всё сканирование и результат
живут только на локальной машине.

## Светофор

- 🔴 **Confirmed** — точная сигнатура именованного чит-клиента из базы (MHUB,
  Doomsday, Cortex, SystemDLC, Novoware, Slinky, Vape, Thunderhack, Prestige,
  Grim Client, Krypton Client).
- 🟡 **Suspicious** — общие/эвристические индикаторы (`KillAura`, `TriggerBot`,
  `AutoAnchor`, `AimAssist` и т.д. из категории «Generic Cheat Module Indicators»).
  Это не подтверждённый детект конкретного клиента, а повод присмотреться —
  такие строки теоретически могут встретиться и в легитимном коде, поэтому
  трактуются мягче.

## Ограничения, которые стоит держать в голове

- Это **сигнатурный** сканер: он находит только то, что явно есть в базе.
  Против неизвестных/свежих читов или клиентов с шифрованием строк в памяти
  (не только в бинарнике на диске) он бессилен без обновления `signatures.json`.
- Сканируются все читаемые committed-регионы, включая память самого JVM (куча,
  метаспейс) и загруженные нативные модули — на процесс с большой кучей (несколько
  гигабайт) полный проход может занять от нескольких секунд до десятков секунд,
  прогресс-бар и статистика в UI показывают ход сканирования в реальном времени.
- Оверлап между чанками рассчитывается от `MaxPatternLength` автомата автоматически,
  поэтому корректность не зависит от того, сколько сигнатур ты добавишь в json.

## Добавление новых сигнатур

Открой `Signatures/signatures.json`, добавь новый объект или дополни существующий:

```json
{
  "Family": "НовыйЧит",
  "Severity": "Confirmed",
  "Patterns": ["уникальная_строка_1", "уникальная_строка_2"]
}
```
